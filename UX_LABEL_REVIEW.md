# UX Label and Wording Review

This document provides a review of the user-facing language in the new event creation workflow, with suggestions for making it more natural and organizer-friendly.

## Guiding Principles

-   **Speak the user's language:** Use terms that an event organizer would naturally use.
-   **Be clear and concise:** Avoid ambiguity and unnecessary words.
-   **Instill confidence:** The language should make the user feel in control and confident in their actions.

## 1. Step Names (in the `StepHeader`)

The current step names are good, but they can be made more active and goal-oriented.

| Current Step Name | Proposed New Name | Rationale |
|---|---|---|
| Event Details | 1. The Event | More personal and direct. |
| Ticket Classes | 2. Tickets | Simpler and more direct. |
| Layout Setup | 3. Seating | "Seating" is a more common and less technical term than "Layout". |
| Assignment | 4. Ticket Assignment | More descriptive of the action being performed. |
| Review & Publish | 5. Review & Publish | This is already clear and action-oriented. No change needed. |

## 2. Page and Section Titles

| Current Title | Proposed New Title | Rationale |
|---|---|---|
| Create New Event | Create an Event | Softer and more conversational. |
| Basic Information | Event Basics | More concise and modern. |
| Schedule | When is your event? | Phrased as a question, which is more engaging. |
| Contact & Policies | Contact & Policies | Clear and concise. No change needed. |
| Fee Configuration | Fees & Currency | Simpler and more direct. |
| Add Ticket Class | Create a Ticket | More direct and action-oriented. |
| Ticket Classes | Your Ticket Options | More user-centric. |
| Layout Setup | Design Your Event Space | More visual and less technical. "Event Space" is more encompassing than "Layout". |
| Review & Publish | Review Your Event | Simpler and more direct. |

## 3. Field Labels and Descriptions

| Current Label/Description | Proposed New Label/Description | Rationale |
|---|---|---|
| Event Title | Event Name | "Name" is slightly more common and less formal than "Title". |
| Hero Image | Cover Image | "Cover Image" is a more modern and widely understood term (e.g., from social media). |
| Promo Video (optional) | Promo Video | Clear. No change. |
| Search Location | Venue Address | More direct and specific. |
| Class Type | Ticket Type | Aligns with the new, simpler terminology. |
| *Description for Class Type* | "Class type drives whether the event needs no layout, seating, table, or mixed setup." | "This determines if you'll need to set up seating for your event." | More direct and focuses on the consequence for the user. |

## 4. Action Labels (Buttons)

| Current Label | Proposed New Label | Rationale |
|---|---|---|
| Create Event & Add Tickets | Save & Add Tickets | Aligns with the step-based flow, where the event is saved as a draft. |
| Add Ticket Class | Add Ticket | Simpler and more direct. |
| Save & Continue | Next: [Next Step Name] | More explicit about what will happen next. For example, "Next: Tickets". This provides a stronger sense of progression. |
| Previous | Back | Simpler and more common. |
| Submit for Approval | Looks Good, Let's Go! | More confident, celebratory, and less formal. It makes the final step feel like a launch, not a submission to a higher authority. |

By adopting this more natural and user-centric language, we can significantly reduce the cognitive load on the organizer and make the platform feel more like an intuitive partner in their event creation process.
