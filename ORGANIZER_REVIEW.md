# Organizer's Review of the Event Platform

This document describes how it feels to use the platform as a non-technical event organizer.

## First Impressions

After signing up, I land on a dashboard. It seems clean, but it's a bit overwhelming. I see stats about revenue and ticket sales, but I haven't even created an event yet. There's a big "+ New Event" button, which is good, but I also see tabs for "Venues", "Payouts", etc. I'm not sure where to start.

## Creating My First Event

I want to create a music concert at a new venue I've just booked. I click "+ New Event". I'm presented with a very long form. It asks for everything all at once: event details, schedule, venue, categories, tags... it feels like a tax form.

I get to the "Venue" dropdown, and my new venue isn't there. I look for a "+ Create New Venue" button, but there isn't one. I'm stuck. I've just spent 10 minutes filling out this form, and now I can't proceed. I feel frustrated and I'm worried I'll lose all the information I've entered.

I have to leave the event creation page and hunt for a way to create a venue. I find a "Venues" tab in the sidebar. I click it and I'm taken to another page where I can create a venue. This is a completely separate process. It feels like I'm using two different applications.

After creating the venue, I have to go back to the dashboard and start creating my event all over again, re-entering all the information I had already typed in.

## Seating Maps are for Experts

When I create my venue, it asks me to set up the seating. I'm presented with a visual builder. This looks powerful, but it's also intimidating. I have to think in terms of "sections", "rows", and "columns". I just want to say "there are 100 seats in the front section and 200 in the back". The tool forces me to think like an architect, not an event planner.

What's more confusing is that after I set up the seating for my venue, I then have to do it *again* for my event. When I go to the event's "Layout" page, it seems to have a copy of my venue's seating map, but it's not clear that changes I make here won't affect my venue's template. I'm worried I'll mess up the seating for future events.

## Technical Jargon

The platform uses words that I don't understand. What's an "inventoryMode"? What's the difference between `seating` and `table` `classType`? Why do I need to know this? I just want to sell tickets for seats at a table. The platform is making me learn its internal language instead of speaking mine.

## Overall Feeling

Using this platform feels like I'm constantly fighting against it. It has powerful features, but they are hidden behind confusing workflows and technical jargon. It doesn't feel like it was designed for someone like me, who is an expert at creating great events, not at configuring software.

It feels rigid and unforgiving. I'm afraid of making a mistake that I can't easily undo. It's a tool for a power user, not for the everyday event organizer.
